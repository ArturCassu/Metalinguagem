getElementById()

O método getElementById() retorna um elemento com um valor especificado. O método getElementById() retornará null se o elemento não existir. O método getElementById() é um dos métodos mais comuns no DOM HTML. Ele é usado quase toda vez que você deseja ler ou editar um elemento HTML. Qualquer id deve ser exclusivo, mas: Se existirem dois ou mais elementos com o mesmo id, getElementById() retornará o primeiro.

Sintaxe:

document.getElementById(elementID)

getElementsByClassName()

O método getElementsByClassName() retorna uma coleção de elementos com um nome de classe especificado(s). O método getElementsByClassName() retorna um HTMLCollection.

Sintaxe:

document.getElementsByClassName(classname)

getElementsByName()

O método getElementsByName() retorna uma coleção de elementos com um nome especificado. O método getElementsByName() retorna um NodeList ativo.

Sintaxe:

document.getElementsByName(name)

getElementsByTagName()

O método getElementsByTagName() retorna uma coleção de todos os elementos com um nome de marca especificado. O método getElementsByTagName() retorna um HTMLCollection. A propriedade getElementsByTagName() é somente leitura. getElementsByTagName("*") retorna todos os elementos no documento.

Sintaxe:

document.getElementsByTagName(tagname)

querySelector()

O método querySelector() retorna o primeiro elemento que corresponde a um seletor CSS. Para retornar todas as correspondências (não apenas a primeira), use querySelectorAll(). Tanto querySelector() quanto querySelectorAll() lançam uma exceção SYNTAX_ERR se o(s) seletor(es) for(em) inválido(s).

Sintaxe:

document.querySelector(CSS selectors)

querySelectorAll()

O método querySelectorAll() retorna todos os elementos que correspondem a um seletor CSS. O método querySelectorAll() retorna um NodeList. O método querySelectorAll() lança uma exceção SYNTAX_ERR se o(s) seletor(es) for(em) inválido(s)

Sintaxe:

document.querySelectorAll(CSS selectors)

Drag

O evento de "drag" em JavaScript permite que elementos HTML sejam arrastados pelo usuário em uma página web. Ele é parte da API de arrastar e soltar (Drag and Drop API), que facilita a interação do usuário com elementos na interface. Quando um elemento é arrastado, o navegador dispara uma série de eventos que podem ser capturados para realizar ações específicas, como atualizar a posição do elemento ou executar alguma lógica de aplicação.

Para usar o evento de drag, geralmente você precisa lidar com três eventos principais:

  • dragstart: Este evento é acionado quando o usuário começa a arrastar um elemento. Aqui, você pode definir o que deve acontecer quando o arraste começa, como configurar dados associados ao arraste (usando setData) e estilizar o elemento durante o arraste.
  • drag: Durante o arraste, o evento drag é disparado continuamente. Você pode usar isso para atualizar a posição do elemento ou realizar outras ações conforme o usuário move o elemento pela página.
  • dragend: Quando o usuário solta o elemento arrastado, o evento dragend é acionado. Aqui você pode limpar quaisquer configurações ou realizar ações finais relacionadas ao arraste.

Esses eventos fornecem uma maneira poderosa de criar interações dinâmicas em suas aplicações web, permitindo que os usuários manipulem elementos de forma intuitiva através do arraste e soltar.
